<h3> 해당 글은 제가 만든 검색 엔진에 대한 리뷰입니다.(검색 엔진이라 불러도 되는가..)

- <h1> 검색 엔진이란? </h1>

 <h3> 사용자의 입력값(검색어, 키워드)에 따른 결과물을 반환해주는 것입니다. </h3>

- <h1> 내가 만든 검색기능 </h1>

<h3> 제가 만든 검색기능은 영화 제목을 검색했을때 해당 영화가 있다면 디테일한 정보를 리턴해주는 방식이였습니다. 맞는 값이 없다면 아무 행동도 하지 않았습니다. 결과적으로 기능의 역활은 했지만 검색기능에 대한 디테일들이 아쉽다고 느껴졌습니다. 해당 디테일들을 보안하기 위해 코드의 리뷰와 개선안을 생각하는 시간을 가졌습니다.</h3>

- <h1> 검색 기능 코드 </h1>

```typescript
// submit 이벤트 발생시 해당 함수가 실행됩니다.

  movie_data.forEach((v: Home_movie) => {
    if (search.trim().toUpperCase() === v.title.trim().toUpperCase()) {
      navigate(`/movie/${v.id}`);
    }
```

<h3> 해당 코드는 API로 받은 Movie의 값들을 순회하며 사용자의 입력값과 같은 제목의 영화를 찾는 기능입니다. 나름의 디테일로 trim()과 toUpperCase()를 넣었으나 다시 보니 별로 쓸모 없어보이며, 너무 급하게 마무리한것 같다는 생각이 들었습니다. 추후 다시 만들게 된다면 해당 방법으로는 만들지 않을 것입니다.</h3>

- <h1> 추후 개선점 </h1>

<h2> 1. 추천검색어 </h2>

<h3> k pop 입력시 검색어 아래에 k popDemonHunters 이런식으로 추천검색어 나오게 하는 방법입니다.</h3>

<h2>2. 검색 실패 </h2>

<h3> 검색 실패시 alert()라던지, 임의로 id값을 줘서 오류페이지를 만들던지 했으면 좋았을거 같습니다. </h3>

<h2> 3. 완전히 일치하지 않아도 검색어가 들어갔다면 해당 영화들 반환 </h2>

<h3> 지금은 완전히 일치해야만 영화를 리턴하는 방식으로 했지만, K라는 단어를 입력시 해당 단어가 들어간 모든 영화를 리턴하는 방식이였으면 좋았을거 같습니다. </h3>
